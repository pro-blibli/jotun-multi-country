How to Create Templates in Talon.One

Campaign templates allow you to create a master campaign configuration that can be reused and deployed across multiple applications with minimal customization. This dramatically reduces the time required to roll out campaigns to multiple geographies.

Step-by-Step Template Creation

1 Build Your Campaign in QA
  • Navigate to your QA application in Talon.One
  • Create a new campaign with all desired rules, conditions, and effects
  • Configure campaign settings: name, description, dates, budgets
  • Thoroughly test the campaign using Talon.One's Rule Tester
  • Validate all conditions trigger correctly and effects execute as expected
2 Configure Campaign Template
  • Once testing is complete, go to Campaign Settings
  • Click "Save as Template" button
  • Provide a descriptive template name (e.g., "Premium Paint Bonus Q2 2025")
  • Add template description explaining the campaign purpose and key parameters
  • Configure condition placeholders
  • Template is now saved and available across all applications in your deployment
3 Deploy Template to Applications
  • Navigate to target application (e.g., Jotun Malaysia)
  • Go to Campaigns → Create New Campaign → "From Template"
  • Select your saved template from the dropdown
  • Input condition placeholder value as per template description
  • Template loads with all rules, conditions, and effects pre-configured
4 Customize for Local Market
  • Update campaign name to include geography (e.g., "Premium Paint Bonus - Malaysia")
  • Adjust start and end dates for local market timing
  • Update budget limits to match geography-specific allocation
5 Review and Activate
  • Use Rule Tester to validate campaign works correctly in this application
  • Review all customizations made for local market
  • Local loyalty manager approves campaign launch
  • Set campaign status to "Active" or schedule for future activation
  • Campaign goes live on configured start date
💡
Template Best Practices
  • Clear Naming: Use descriptive template names that indicate campaign type and timeframe
  • Documentation: Add comprehensive descriptions explaining campaign logic and customization points
  • Version Control: Include version numbers or dates in template names for tracking iterations
  • Regular Updates: Keep templates updated based on learnings from deployed campaigns

Global Campaign Rollout Strategies

Talon.One offers three strategic approaches for rolling out campaigns across multiple geographies, each with different levels of automation and control.

1
MANUAL APPROACH

Duplicate Campaigns from QA

Manually duplicate campaigns from the QA application across multiple live applications. This approach gives maximum control and visibility at each step.

Process:

  • Create and test campaign in QA application
  • Navigate to campaign → Actions → Duplicate Campaign
  • Select target application (e.g., Malaysia, Vietnam, Philippines)
  • Campaign copies with all rules, conditions, and effects intact
  • Adjust dates, budgets, and currency-specific values per geography
  • Local loyalty manager reviews and activates
  • Repeat for each geography

✅ Advantages:

  • No technical skills required
  • Complete visibility and control at each step
  • Easy to customize for local market requirements
  • No risk of automation errors
  • Application/country specific loyalty manager can duplicate and change values to activate campaign in minutes

⚠️ Considerations:

  • Time-consuming with many geographies (17+ countries) if this is handled by a group level loyalty manager
  • Repetitive manual work for each application if this is handled by a group level loyalty manager
  • Risk of human error in duplication process
  • Difficult to maintain consistency across all markets
2
TEMPLATE APPROACH

Templatize Base Campaign

Create a base campaign template in the QA application and use Talon.One's campaign templates feature to roll out campaigns efficiently. Each geography's loyalty manager activates the template in their application.

Process:

  • Create and thoroughly test campaign in QA application
  • Save campaign as a template with placeholder values for geography-specific parameters
  • Distribute template to all geography applications
  • Local loyalty manager opens template in their application
  • Manager customizes dates, budgets, and local parameters
  • Manager activates campaign when ready
  • Template remains available for future use

✅ Advantages:

  • Faster than manual duplication across many geographies
  • Maintains consistency with base template structure
  • Local loyalty managers retain control and flexibility
  • Reusable templates for recurring campaigns (e.g., quarterly promotions)
  • Minimal technical skills required

⚠️ Considerations:

  • Requires manual activation per geography
  • Each loyalty manager needs to remember to activate
  • Risk of inconsistent launch timing across geographies
  • Template updates don't automatically propagate to existing campaigns
3
AUTOMATED APPROACH

API-Driven Campaign Deployment

Use Talon.One's "Create Campaign via Template" API to programmatically roll out campaigns across all geographies. This can be triggered via an internal UI or an MCP (Model Context Protocol) connector.

Process:

  • Create and test campaign template in QA application
  • Define geography-specific parameters in configuration file (dates, budgets, currencies)
  • Build internal UI or configure MCP connector to call Talon.One API
  • Select target geographies from UI
  • Click "Deploy Campaign" button
  • API automatically creates campaigns in all selected applications
  • Local loyalty managers receive notification to review and activate
API Example: Create Campaign via Template
POST /v1/applications/{applicationId}/campaigns/from_template

{
  "templateId": 789,
  "name": "Premium Paint Bonus - Malaysia",
  "startTime": "2025-06-01T00:00:00Z",
  "endTime": "2025-06-30T23:59:59Z",
  "budgetLimit": 50000,
  "state": "disabled"  // Start disabled for local manager review
}

✅ Advantages:

  • Highly scalable - deploy to 17+ countries in minutes
  • Eliminates manual repetition entirely
  • Consistent campaign structure across all geographies
  • Reduces human error in campaign setup
  • Version-controlled campaign configurations
  • Can integrate with approval workflows and scheduling systems
  • Fast response to market opportunities

⚠️ Considerations:

  • Requires development resources for initial setup
  • Higher initial time investment to build UI or MCP connector
  • Need API knowledge and error handling implementation
  • Requires ongoing maintenance of automation scripts
  • Must handle API rate limits for bulk operations

Recommendation for Jotun

Start with Approach 2 (Template Approach) for your initial rollout across 17+ countries. This provides a good balance between efficiency and control, allowing local loyalty managers to maintain oversight while significantly reducing deployment time.

Evolve to Approach 3 (API-Driven) once you have 6+ months of operational experience and are launching campaigns frequently (monthly or more). The upfront development investment will pay dividends with increased campaign velocity and reduced manual work for your loyalty managers.

Automating with MCP & Template API

For organizations seeking maximum efficiency, an internal MCP (Model Context Protocol) connector integrated with Talon.One's Template API can fully automate the campaign deployment process across all geographies.

🤖
What is MCP?

Model Context Protocol (MCP) is an open protocol that enables AI assistants like Claude to securely connect with external systems and APIs. By building an internal MCP connector, you can allow your team to deploy campaigns across multiple geographies using natural language commands.

How MCP + Template API Works

1
Natural Language Request

Loyalty manager: "Deploy Premium Paint campaign to all ASEAN markets"

→
2
MCP Processing

MCP connector interprets request and identifies target applications

→
3
API Calls

Automated API calls to Talon.One Template API for each geography

→
4
Campaigns Created

Campaigns deployed to Malaysia, Vietnam, Philippines, Thailand, Indonesia

Benefits of MCP Automation

⚡

Instant Deployment

💬

Intelligent Parameter Mapping

Batch Operations

Audit Trail

🛡️

Built-in Safeguards

Getting Started with MCP Automation

Building an internal MCP connector requires development resources but provides significant long-term efficiency gains. The typical implementation timeline is 2-4 weeks for a basic connector, with ongoing enhancements based on team feedback.

Key Components Needed:

  • MCP server implementation (handles natural language processing and API orchestration)
  • Talon.One Template API integration (authentication, endpoint configuration)
  • Geography mapping configuration (defines application IDs, currencies, timezones, budget rules)
  • Permission and approval workflow logic (ensures proper access control)
  • Logging and monitoring infrastructure (tracks all automated deployments)

ROI Consideration: If your organization deploys campaigns monthly or more frequently across 10+ geographies, MCP automation typically pays for itself within 3-6 months through time savings alone.
